# AdvHarvest Autopilot

Система, которая **находит насущные потребности людей в интернете** и готовит способ их удовлетворить. Деньги — следствие пользы, не самоцель.

## Цикл

```
Discover → Score → Fulfill → Earn
```

1. **Discover** — сканирует открытые источники:
   - Hacker News Ask
   - Stack Overflow / Stack Overflow на русском
   - GitHub Issues
2. **Score** — срочность, ясность, вовлечённость, категория
3. **Fulfill** — выбирает способ закрыть боль и пишет черновик ответа
4. **Earn** — ценность учитывается в ledger после вашего approve

## Способы удовлетворить потребность

| Режим | Когда |
|---|---|
| **Гайд / ответ** | Можно закрыть понятным пошаговым решением |
| **Микро-инструмент** | Люди ищут tool / шаблон / скрипт |
| **Услуга под ключ** | Нужен исполнитель, можно сделать самим |
| **Подбор исполнителя** | Сами не делаем — передаём лид за комиссию |

Human-in-the-loop: публикация ответа и «закрытие» — только после `Approve` / `Fulfill`.

## Быстрый старт

```bash
npm run demo      # живой скаут → планы → approve → fulfill
npm run cycle     # один проход по сети
npm start         # дашборд http://localhost:3847
```

Зависимостей нет — Node.js ≥ 18. Результаты скаута кэшируются в `data/needs-cache.json`.

## Структура

```
src/scout.js       — поиск потребностей в сети
src/needs.js       — score + план удовлетворения
src/engine.js      — цикл, ledger, learn
src/store.js       — JSON-хранилище
src/server.js      — API + дашборд
public/            — UI
data/signals.json  — старые демо-сигналы (ниша стройки)
```

## Принцип

Сначала закрываем реальную боль человека из открытого запроса.  
Монетизация (гайд→продукт, услуга, комиссия) — второй шаг.
